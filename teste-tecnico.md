Teste Técnico - Desenvolvedor React Native
SuitPay – Fintech

Sobre o Desafio
Bem-vindo(a) ao teste técnico da SuitPay! Queremos avaliar suas habilidades em React Native através do desenvolvimento de um aplicativo de lista de tarefas (To-Do List).

Objetivo
Desenvolver um aplicativo mobile de gerenciamento de tarefas que demonstre seus conhecimentos em React Native, gerenciamento de estado, componentização, persistência de dados e boas práticas de código.

 Requisitos Funcionais Obrigatórios
1. Listagem de Tarefas
   - Exibir lista de todas as tarefas cadastradas
   - Mostrar status visual da tarefa (completa/incompleta)
   - Exibir data de criação de cada tarefa

2. Adicionar Tarefa
   - Campo de input para título da tarefa
   - Botão para adicionar nova tarefa
   - Validação: não permitir tarefas vazias

3. Marcar como Concluída
   - Permitir alternar status da tarefa (completa/incompleta)
   - Indicação visual clara do status

4. Excluir Tarefa
   - Botão/ação para remover tarefa da lista
   - Confirmação antes de excluir (alert ou modal)

5. Persistência de Dados
   - Utilizar AsyncStorage para salvar as tarefas
   - Tarefas devem permanecer após fechar o app

6. Contador de Tarefas
   - Exibir total de tarefas
   - Exibir quantas estão concluídas
   - Exibir quantas estão pendentes

 Diferenciais (Opcionais)
- Filtros: possibilidade de filtrar tarefas (todas/ativas/concluídas)
- Edição: permitir editar o título de uma tarefa existente
- Categorias: adicionar categorias ou tags às tarefas
- Prioridades: sistema de prioridade (alta/média/baixa)
- Busca: campo de busca para filtrar tarefas por texto
- Animações: adicionar animações nas transições e interações
- Dark Mode: suporte a tema claro e escuro

Requisitos Técnicos Obrigatórios
- React Native CLI (não utilize Expo)
- Hooks do React (useState, useEffect, etc.)
- AsyncStorage para persistência
- Código organizado em componentes

Desejáveis
- TypeScript
- Context API ou outra solução de gerenciamento de estado
- Styled Components ou biblioteca de estilização
- Componentização clara e reutilizável

Estrutura de Entrega
 O que enviar:

1. Repositório Git (GitHub/GitLab/Bitbucket)
   - Código-fonte completo
   - Commits organizados e descritivos
   - .gitignore configurado

2. README.md contendo:
   - Descrição do projeto
   - Instruções para rodar o app
   - Tecnologias utilizadas
   - Funcionalidades implementadas
   - Screenshots ou GIF do app funcionando
   - Desafios encontrados e soluções aplicadas

3. Estrutura de pastas sugerida:
/src
  /components
  /screens
  /services
  /utils
  /styles
  /types (se usar TypeScript)

 Como Começar
1. Crie um novo projeto React Native CLI
2. Configure o ambiente de desenvolvimento
3. Instale as dependências necessárias
4. Implemente as funcionalidades obrigatórias primeiro
5. Adicione diferenciais se houver tempo
6. Teste em dispositivos iOS e/ou Android
7. Documente no README
8. Faça commit e push para seu repositório
